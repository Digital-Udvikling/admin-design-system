Thin React wrappers that emit the same class names as the vanilla CSS.

## Install

```bash
npm install @aortl/admin-react react react-dom
```

## Import styles + components

```tsx
import "@aortl/admin-react/styles.css";
import { Button, Card, Input } from "@aortl/admin-react";

export function SignIn() {
  return (
    <Card>
      <Card.Body>
        <Card.Title>Sign in</Card.Title>
        <Input placeholder="Email" />
        <Input type="password" placeholder="Password" />
        <Card.Actions>
          <Button variant="primary">Sign in</Button>
          <Button variant="ghost">Cancel</Button>
        </Card.Actions>
      </Card.Body>
    </Card>
  );
}
```

## Add icons (optional)

The design system recommends [Tabler Icons](https://tabler.io/icons) as the icon library — see [Icons](../../theme/icons/) for the rationale, sizing convention, and usage patterns. Install the typed React package:

```bash
npm install @tabler/icons-react
```

Then import icons by name — bundlers will tree-shake unused ones:

```tsx
import { IconPlus, IconTrash } from "@tabler/icons-react";

<Button variant="primary">
  <IconPlus size={16} aria-hidden />
  New order
</Button>;
```

Components like `<Button>`, `<Menu.Item>`, `<Alert>`, and `<Sidebar.Icon>` already lay icons out — no wrapper required.

## Notes

- Styles come from one CSS file — see [Customize](../../theme/customize/) to override design tokens.
- `<Button>`, `<Input>`, and `<Field>` wrap [Base UI](https://base-ui.com) primitives — keyboard, focus, and validation accessibility are handled for you.
